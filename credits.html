Security Considerations

One major security concern is the way the add event and edit event pages are coded. If an attacker was to get onto the admin page of the site, they could do some serious damage to the database data. At the moment all an attacker needs to do it write out an SQL script into one of the input boxes to manipulate the code. From here an attacker could freely alter data aswell as delete massive chunks causing massive issues for data loss as well as downtime for the website. Similarly the collumn names are exposed when moving from page to page between the edit events list page and the edit page itself. This could allow an attack to easily attack the database directly via a SQLijection attack.

Another security concern is the lack of a login system for the admin. This leaves the site completely exposed to anyone who visited the admin page. My advice would to get a password protected user system in place to restrict access to this area of the website to users who have the right credentials to log in. By doing this you increase the difficulty for an attack to gain access to the sensitive information you hold in your database.
